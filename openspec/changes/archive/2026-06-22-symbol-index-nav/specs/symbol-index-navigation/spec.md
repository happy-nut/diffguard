## ADDED Requirements

### Requirement: 블록 없는 선언 인덱스
소스 심볼의 선언 위치 인덱스가 UI를 블록하지 않고 빌드되어야 한다(SHALL). 인덱스가 준비되면 선언 조회는 전 파일 전수 스캔 없이 인덱스로 해석되어야 한다(SHALL).

#### Scenario: 인덱스가 백그라운드에서 빌드된다
- **WHEN** 리뷰가 로드될 때
- **THEN** 선언 인덱스가 메인 스레드를 블록하지 않고(유휴 시간/비동기) 빌드된다

#### Scenario: 인덱스로 즉시 해석
- **WHEN** 인덱스가 준비된 상태에서 사용자가 단어의 선언으로 점프할 때
- **THEN** 선언 위치가 전 파일 전수 스캔 없이 인덱스에서 조회된다

### Requirement: 커서 단어의 선언으로 네비게이션
사용자는 소스 뷰와 diff 뷰 **모두에서** `Cmd/Ctrl+B`로 커서 위치 단어의 선언부로 점프할 수 있어야 한다(SHALL). 소스 뷰에서는 `Cmd/Ctrl+Down`도 동일하게 동작해야 한다(SHALL).

#### Scenario: 소스 뷰에서 Cmd+B로 선언 점프
- **WHEN** 소스 뷰에서 캐럿이 어떤 식별자 위에 있고 사용자가 `Cmd/Ctrl+B`를 누를 때
- **THEN** 그 식별자의 선언이 있는 파일·줄로 캐럿과 함께 이동한다

#### Scenario: diff 뷰에서 Cmd+B로 선언 점프
- **WHEN** diff 뷰에서 diff 캐럿이 어떤 식별자 위에 있고 사용자가 `Cmd/Ctrl+B`를 누를 때
- **THEN** 소스 뷰로 전환되어 그 식별자의 선언이 있는 파일·줄로 이동한다

#### Scenario: 소스 뷰의 Cmd+Down도 동일하게 동작
- **WHEN** 소스 뷰에서 사용자가 `Cmd/Ctrl+Down`을 누를 때
- **THEN** `Cmd/Ctrl+B`와 동일하게 커서 단어의 선언으로 이동한다

### Requirement: 인덱스 미스 시 스캔 폴백
인덱스에 선언이 없거나 인덱스가 아직 준비되지 않았으면 기존 전수 스캔 방식으로 폴백해야 한다(SHALL). 인덱스 도입으로 기존에 찾던 선언을 못 찾게 되어서는 안 된다(MUST NOT).

#### Scenario: 인덱스에 없는 심볼
- **WHEN** 어떤 식별자가 인덱스에 없는 상태에서 사용자가 선언으로 점프할 때
- **THEN** 전수 스캔 폴백으로 선언을 찾아 이동하거나, 선언 자체가 없으면 아무 일도 일어나지 않는다

#### Scenario: 인덱스 빌드 완료 전
- **WHEN** 인덱스가 아직 빌드되지 않은 상태에서 사용자가 선언으로 점프할 때
- **THEN** 전수 스캔 폴백으로 동작한다
